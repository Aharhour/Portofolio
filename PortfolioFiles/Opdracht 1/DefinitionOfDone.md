# Definition of Done

Een user story wordt als "Done" beschouwd wanneer aan alle onderstaande criteria is voldaan:

## Algemene criteria

1. **Code is geschreven en werkend** - De functionaliteit is volledig geimplementeerd volgens de acceptatiecriteria van de user story.
2. **Code is gereviewed** - De code is bekeken en goedgekeurd (self-review of peer review).
3. **Geen bekende bugs** - Er zijn geen openstaande bugs gerelateerd aan de user story.
4. **Responsive** - De functionaliteit werkt correct op zowel mobiel (375px) als desktop (1440px).
5. **Getest** - De functionaliteit is handmatig getest en werkt zoals verwacht.
6. **Gedeployed** - De code is succesvol gemerged naar de main branch en draait op de productieomgeving.

## Technische criteria

1. **Geen console errors** - De browser console toont geen errors gerelateerd aan de nieuwe functionaliteit.
2. **Authenticatie** - Beveiligde routes zijn alleen toegankelijk voor geautoriseerde gebruikers.
3. **Formuliervalidatie** - Alle invoervelden hebben passende validatie met duidelijke foutmeldingen.
4. **Performance** - Pagina's laden binnen 2 seconden.

## Documentatie criteria

1. **User story is bijgewerkt** - De status van de user story is aangepast in de backlog.
2. **Acceptatiecriteria afgevinkt** - Alle acceptatiecriteria van de story zijn afgevinkt.
